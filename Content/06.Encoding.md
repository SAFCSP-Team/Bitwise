# Encoding

Computers store and process all data in binary form (0s and 1s). However, humans work with characters such as letters, numbers, and symbols. To bridge this gap, we use **encoding**. Encoding is the process of representing real-world information, such as characters, as binary values so that computers can store, transmit, and process them.

Different types of data require different encoding systems. Numbers can be converted directly into binary, but characters need standardized mappings. Two of the most widely used character encoding standards are **ASCII** and **Unicode**, which define how letters, digits, punctuation, and symbols are represented as binary patterns.

## ASCII

ASCII (American Standard Code for Information Interchange) is one of the earliest character encoding standards. It represents characters using **7 bits**, providing a total of **128 possible values**, including letters, digits, punctuation, and control characters.

Examples:

- `A` → 65 → 01000001
- `a` → 97 → 01100001
- `0` → 48 → 00110000


> [!TIP]
> The ASCII table is a table displaying all characters encoding in binary ([Example of the ASCII table](https://www.ascii-code.com/)).


## Unicode

Unicode is a modern encoding standard designed to represent characters from **many languages and writing systems** around the world. Unlike ASCII, which is limited to 128 characters, Unicode supports **thousands of characters**, including emojis and characters from different languages.

Unicode uses several encoding formats, such as:
- UTF-8
- UTF-16
- UTF-32

The UTF‑8, UTF‑16, and UTF‑32 are all Unicode encoding formats. They store the same characters but with different numbers of bytes.

> [!NOTE]
> ASCII is actually a subset of Unicode. This means that standard ASCII characters have the same binary representation in UTF-8.
