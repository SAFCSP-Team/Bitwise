# Masks

A **bit mask** is a binary pattern used together with bitwise operations to manipulate specific bits in a value.  
Masks allow programmers to **set, clear, toggle, or check** individual bits without affecting the other bits.

Bit masks are commonly used in low-level programming, flags, permissions, hardware control, and performance-critical systems.


## Setting a Bit

Setting a bit means forcing a specific bit to **1**.  
This is usually done using the **OR (|)** operation with a mask.

Example:
```
Value: 0101
Mask: 0010

Result: 0111
```

## Clearing a Bit

Clearing a bit means forcing a specific bit to **0**.  
This is done using **AND (&)** with an inverted mask.

Example:
```
Value: 1101
Mask: 1011

Result: 1001
```

## Toggling a Bit

Toggling a bit means flipping its value (**0 → 1** or **1 → 0**).  
This is done using the **XOR (^)** operation.

Example:
```
Value: 0101
Mask: 0010

Result: 0111
```

## Checking a Bit

To check whether a bit is **1 or 0**, use the **AND (&)** operation with a mask.

Example:
```
Value: 1010
Mask: 0010

Result: 0010
```

If the result is **non-zero**, the bit was set.

> [!NOTE]
> Bit masks allow precise control over individual bits, which makes them very useful for working with flags, hardware registers, permissions, and compact data representations.
