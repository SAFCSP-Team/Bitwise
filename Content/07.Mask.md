# Masks
In a sequence of bits, a **mask** allows programmers to set, clear, toggle, or check individual bits without affecting the others. This is achieved by using a **bit mask**, which is a binary pattern used together with bitwise operations to isolate and manipulate specific bits in a value. Bit masks are commonly used in low-level programming, flags, permissions, hardware control, and performance-critical systems.

In the following sections, we will explore how to use the mask to set, clear, toggle, and check individual bits.

## Set

Setting a bit means forcing a specific bit to have the value of **1**. This is usually done using the **OR** operation with a mask.

**Example**

Setting the second bit:

```
Value:  0101
Mask:   0010

Result: 0111
```

## Clear

Clearing a bit means forcing a specific bit to be **0**. This is done using the **AND** operation with an inverted mask (the target bit is 0, all others are 1).

**Example**

Clearing the third bit:
```
Value: 1101
Mask: 1011

Result: 1001
```

## Toggle

Toggling a bit means flipping its value **0 → 1** and **1 → 0**. This is done using the **XOR** operation.

**Example**

Toggling the second bit:
```
Value:  0101
Mask:   0010

Result: 0111
```

## Check

To check whether a bit is **1** or **0**, use the **AND** operation with a mask.

**Example**

Checking the second bit:
```
Value:  1010
Mask:   0010

Result: 0010
```

If the result is **non-zero**, the bit is set (the value is **1**).

> [!NOTE]
> Bit masks allow precise control over individual bits, which makes them very useful for working with flags, hardware registers, permissions, and compact data representations.
